# ReppedBooth Bug Reports

This is the bug report repository for the Repped Labs QA Fundamentals Circle.

## What this repo is

This is where you file bug reports for the ReppedBooth practice app. Your reports go here as GitHub Issues starting in Sprint 2.

This repo was pre-seeded with bug reports to simulate joining a team that already started testing this app. Those reports are labeled `seed`. Yours will appear alongside them.

## The practice app

Open ReppedBooth here: [ReppedBooth — link in cohort details]

## How to file a bug report

1. Click **Issues** in the top navigation.
2. Click **New issue**.
3. Fill in every field in the template. Do not leave fields blank.
4. Apply the label for the version you are testing: `v1`, `v2`, or `v3`.
5. Apply the label for the bug type (see label descriptions in the Labels tab).
6. Click **Submit new issue**.

Your report is submitted when it appears in the Issues list with your username as the author.

## How to filter by version

1. Click **Issues**.
2. Click the search bar.
3. Type `label:v1` (or `v2` or `v3`) and press Enter.

To see closed issues too: type `is:closed label:v1`.
